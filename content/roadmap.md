# TKO Roadmap

Last updated: 2026-06-15

## Roadmap Rule

The next 90 days should optimize for revenue, marketability, proof of value, and implementation readiness.

Do not build platform infrastructure before paid demand proves a repeated need.

TIF content work should use the documented operating model:

```text
Knowledge -> Insight -> Deliverable -> Channel Package -> Publication -> Measurement
```

This is a planning rule, not a build authorization. Prioritize proof and revenue-producing
deliverables before generic content infrastructure.

## 0-30 Days

| Priority | Work | Why | Exit Criteria |
|---:|---|---|---|
| P0 | Publish TKO offer one-pager for Operational Intelligence Diagnostic. | Needed to sell. | One concise sales page or PDF complete. |
| P0 | Publish Rachel operational intelligence case study. | Best proof asset. | Case study includes problem, system, evidence, current limits. |
| P0 | Start outreach to healthcare / health-tech network. | Fastest route to paid diagnostic. | 20 targeted conversations or asks. |
| P0 | Activate Rachel outreach drafts if operator-approved. | Turns built capability into proof. | First batch reviewed; approval/send metrics tracked. |
| P1 | Create weekly Rachel outcome scorecard. | Revenue proof needs outcomes. | Leads, contacted, replies, appointments, showings, contracts, closings, revenue, drafts. |
| P1 | Rewrite shared intelligence vocabulary. | Needed before any platform reuse discussion. | Definitions with Rachel examples for Observation, Fact, Recommendation, Source, Confidence, Comparison. |
| P1 | Rebaseline TIF content vocabulary. | Prevents asset-only thinking and supports demand generation. | Docs distinguish Knowledge, Insight, Deliverable, Channel Package, Publication, and Measurement. |

## 31-60 Days

| Priority | Work | Why | Exit Criteria |
|---:|---|---|---|
| P0 | Sell or seriously validate one Operational Intelligence Diagnostic. | Revenue proof. | Signed diagnostic or documented rejection pattern from qualified buyers. |
| P0 | Package Relationship Intelligence as a capability brief. | Differentiated offer. | Brief includes architecture, screenshots/examples, implementation path. |
| P1 | Complete referral lifecycle if used in demos/offers. | Prevents partial capability from looking unfinished. | Referral sent/accepted/closed states and suppression. |
| P1 | Improve behavioral signal consumption V1. | Highest-value intelligence gap. | Email click/open/reply signals affect queue/gap surfacing. |
| P1 | Package reporting modernization template. | Supports CRE/reporting conversations without claiming report engine. | Executive report template plus source/confidence model. |

## 61-90 Days

| Priority | Work | Why | Exit Criteria |
|---:|---|---|---|
| P0 | Convert diagnostic into implementation or advisory retainer. | Revenue expansion. | Proposal for $50k+ implementation or monthly advisory. |
| P0 | Produce second case study or proof artifact. | Market trust. | Healthcare-adapted diagnostic example or Rachel follow-up metrics. |
| P1 | Decide whether CRE/reporting wedge has proof. | Avoid architecture-first build. | Paid report, proposal, or explicit no-go evidence. |
| P2 | Define extraction gates for shared services. | Prevent premature platform work. | Objective thresholds for reuse: 2 paid workflows, repeated duplication, measurable ROI. |

## Explicit Deferrals

Do not build in the next 90 days unless a paying customer requires it:

- Multi-tenant SaaS.
- Intelligence graph runtime.
- CRM integrations.
- Model gateway.
- Generic report engine.
- Generic content engine.
- Autonomous outreach.
- Marketplace/channel integrations.
